Foundations — Why Transformers, and What Goes In

FOUNDATIONAL

Before 2017, sequence models (like RNNs and LSTMs) processed text one token at a time, in order. That made them slow to train and forgetful over long sentences — by the time an LSTM reached word 40, it had a fuzzy memory of word 2. The 2017 paper "Attention Is All You Need" proposed something radical: drop recurrence entirely, and let every token look directly at every other token, all at once.

Analogy Imagine reading a mystery novel one word at a time through a mail slot, trying to remember every clue by memory (that's an RNN). A transformer instead spreads the entire page on a table and lets you glance between any two words instantly — "who does 'he' refer to?" is answered by looking, not remembering.

What actually enters the model

A sentence like "The cat sat" can't be fed to a neural network as text. It goes through two conversions:

1. Tokenization + Embedding Each word (or sub-word piece) is mapped to an ID, then to a learned vector — typically 512 to 12,000+ numbers — that captures its meaning. Similar words end up with similar vectors: king and queen sit closer together than king and banana.
2. Positional Encoding Since attention has no built-in sense of order, a positional signal (often sine/cosine waves of different frequencies) is added to each embedding so the model knows "cat" was word 2, not word 200.
Every token → embedding vector → + positional signal → model input

Comparison: RNN vs Transformer

PropertyRNN / LSTMTransformer
Processing orderSequential, one token at a timeParallel, all tokens at once
Long-range memoryDegrades over distanceDirect token-to-token access
Training speed on GPUsSlow (can't parallelize time steps)Fast (matrix ops parallelize well)

⚠ Common misconception

"Transformers understand word order because they read left to right." False — a transformer sees the whole sequence simultaneously. Order is only known because of the positional encoding added at the input; remove it and the model would treat a sentence as a bag of words.

Practical exercise In your own words, explain to a beginner why an RNN struggles with the sentence: "The trophy didn't fit in the suitcase because it was too big." (Hint: what does "it" refer to, and how far back must the model look?)
An RNN must carry the meaning of "trophy" and "suitcase" forward through every intermediate word before deciding what "it" refers to — that signal weakens over distance. A transformer instead lets "it" directly attend back to both "trophy" and "suitcase" in one step and weigh which fits the context of "too big," regardless of distance.

Key takeaways

  • Transformers process all tokens in parallel, unlike sequential RNNs
  • Each token becomes a vector via embedding, then gets positional information added
  • Removing positional encoding would make the model blind to word order

Self-Attention Mechanics — Query, Key, Value

CORE MECHANISM

Self-attention is the heart of the transformer. For every token, the model asks: "which other tokens matter to me, and how much?" It answers this using three learned projections of each token's embedding: Query (Q), Key (K), and Value (V).

Analogy Think of a library. Your Query is the question you bring ("I need info about cats"). Every book has a Key on its spine summarizing its content. You compare your query to every key to find relevance, then you actually read the Value — the book's content — from the best matches, blending information proportional to relevance.

The core formula

Attention(Q, K, V) = softmax( Q·KT / √dk ) · V

Step by step:

  1. Q·KT — dot-product every query against every key → raw relevance scores
  2. ÷ √dk — scale down so scores don't explode as vector size grows (keeps gradients stable)
  3. softmax — turn scores into a probability distribution that sums to 1 (who gets how much attention)
  4. · V — blend the Value vectors using those probabilities as weights

Worked example: "The cat sat"

Softmax turns raw relevance scores into attention weights that sum to 100%

Darker = higher attention weight from selected token
Example: pronoun resolution In "The dog chased the ball because it was fast", the query for "it" produces high attention on "dog" if "fast" describes the dog's speed — the model learns this from patterns across billions of sentences, not from grammar rules.
Example: same word, different meaning In "river bank" vs "savings bank", self-attention lets "bank" pull different context from its neighbor, producing a different final representation each time — this is how transformers handle polysemy.

⚠ Common misconception

"Q, K, and V are three different pieces of the sentence." No — they're three different linear projections of the same input embedding, using three separate learned weight matrices (WQ, WK, WV). Every token generates its own Q, K, and V.

Practical exercise Given raw attention scores [2.0, 6.0, 2.0] for a token attending to 3 words, roughly which word will dominate the output after softmax, and why doesn't the exact ratio matter as much as the relative gap?
The second word (score 6.0) will dominate — softmax is exponential, so a gap of 4 becomes a huge probability gap (roughly 98% vs 1% each for the others). This is why attention tends to be "peaky": once one token is clearly more relevant, softmax amplifies that difference rather than blending everything evenly.

Key takeaways

  • Every token produces a Query, Key, and Value via three learned weight matrices
  • Attention weights come from softmax(Q·Kᵀ / √d_k), always summing to 1 per token
  • The output for each token is a weighted blend of all Value vectors in the sequence

The Full Block — Multi-Head Attention, Residuals, FFN

APPLICATION

A single attention pass captures one "type" of relationship. Real transformers run multiple attention heads in parallel, each learning to focus on different relationships — one head might track grammar, another might track coreference, another long-range topic relevance.

Analogy Multi-head attention is like having several editors review the same paragraph simultaneously: one checks grammar, one checks factual consistency, one checks tone. Each produces separate notes (heads), then a final editor merges them into one cohesive revision (the output projection).
One full transformer block: multi-head attention → residual + norm → feed-forward → residual + norm

Why residual connections and layer norm matter

Residual connections Each sub-layer's output is added back to its input (x + Sublayer(x)) rather than replacing it. This lets gradients flow directly through many stacked layers (12, 48, 96+) without vanishing — the model can always fall back to "pass the input through unchanged" if a layer isn't helpful yet.
Layer normalization Rescales each token's vector to have stable mean/variance before the next step. Without it, values can drift to extreme magnitudes across dozens of stacked layers, destabilizing training.

The Feed-Forward Network (FFN)

After attention mixes information across tokens, the FFN processes each token independently — typically expanding to 4× the model's dimension, applying a non-linearity (like GELU), then projecting back down. This is where much of the model's "knowledge" and pattern recognition is thought to live.

Interactive: how many heads should this model use?
Heads: 8
8 heads: a balanced default (used in the original "Attention Is All You Need" base model) — enough diversity of relationship-types without excessive compute.

⚠ Common misconception

"More heads always means a smarter model." Not quite — the total attention dimension is usually fixed and split across heads, so more heads means each head is narrower. There's a sweet spot; too many heads can dilute each head's capacity.

Practical exercise A colleague removes the residual connections from a 24-layer transformer to "simplify the architecture." Predict what happens during training, and why.
Training would likely become unstable or fail to converge. Without residual paths, gradients must pass through 24 stacked non-linear transformations during backpropagation, causing vanishing/exploding gradients. Early layers would receive almost no useful learning signal, and the model would struggle to improve beyond a shallow-network baseline.

Key takeaways

  • Multi-head attention runs several attention "views" in parallel, then merges them
  • Residual connections let gradients flow through deep stacks of layers
  • Layer norm keeps activations stable across many stacked blocks
  • The feed-forward network processes each token independently after attention mixes context

Architectures & Mastery — Encoder, Decoder, and Real Models

MASTERY

The original transformer paper described an encoder-decoder design for translation. Since then, the field has largely split into three families, each keeping the same core attention block but arranging it differently.

Encoder-only (e.g. BERT) Every token can attend to every other token, including ones that come after it ("bidirectional"). Great for understanding tasks: classification, embeddings, search.
Decoder-only (e.g. GPT, Claude) Each token can only attend to itself and earlier tokens ("causal masking") — because the model must predict the next token without seeing the future. This is the dominant design for modern generative LLMs.
Encoder-decoder (e.g. T5, original Transformer) An encoder reads the full input (like a source sentence), and a decoder generates output token by token while attending back to the encoder's representations via "cross-attention." Common in translation and summarization.

Causal masking, visualized

Rows = query token, columns = key token. Green = visible, dark = masked out
Analogy An encoder is like an editor who gets to read the whole manuscript before commenting on any single sentence. A decoder is like a live novelist dictating a story out loud — each new sentence can only reference what's already been said, never what comes next.

Comparison table

TypeAttention directionBest forExamples
Encoder-onlyBidirectionalClassification, embeddingsBERT, RoBERTa
Decoder-onlyCausal (left-to-right)Text generation, chatGPT, Claude, Llama
Encoder-decoderBidirectional + cross-attnTranslation, summarizationT5, original Transformer

⚠ Common misconception

"GPT-style models can't understand context as well as BERT because they're one-directional." In practice, causal decoder-only models scaled to huge sizes (GPT-4, Claude) dramatically outperform bidirectional encoders on almost every task — architecture direction matters less than scale, data, and training objective at this point.

Practical exercise You're designing a customer-support chatbot that must generate free-form replies. Which architecture family would you choose, and what masking strategy does it require?
A decoder-only architecture with causal masking is the standard choice — it naturally supports open-ended, token-by-token generation, matches how modern chat LLMs (GPT, Claude) are built, and doesn't require a separate encoder since there's no fixed source-to-target mapping like in translation.

Key takeaways

  • Encoder-only = bidirectional, best for understanding tasks (BERT)
  • Decoder-only = causal masking, dominant for modern generative LLMs (GPT, Claude)
  • Encoder-decoder = full source read + cross-attention, best for translation-like tasks
  • At scale, decoder-only causal models have proven remarkably capable across nearly all tasks

Reference

Cheat Sheet

Core formula

Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V

Pipeline (decoder-only, e.g. GPT/Claude style)

Token → Embedding → + Positional Encoding → [Masked Multi-Head Self-Attention → Add & Norm → Feed-Forward → Add & Norm] × N layers → Output projection → Softmax over vocabulary

Vocabulary

  • Q/K/V — Query / Key / Value projections of a token
  • Multi-head — several parallel attention computations, concatenated
  • Residual connection — x + Sublayer(x), preserves gradient flow
  • Layer norm — normalizes activations for training stability
  • Causal mask — prevents attending to future tokens (decoder-only models)
  • Cross-attention — decoder attends to encoder outputs (encoder-decoder models)

Summary Notes

Transformers replaced sequential RNNs with parallel self-attention, letting every token directly relate to every other token via learned Query/Key/Value projections. Multiple attention heads capture different relationship types simultaneously; residual connections and layer normalization keep deep stacks (dozens of layers) trainable; feed-forward layers process each token's representation independently. Architecture families (encoder-only, decoder-only, encoder-decoder) differ mainly in attention direction and masking, with decoder-only causal models now dominating large-scale generative AI.
